# SISTEMAS DISTRIBUÍDOS

- **Heterogêneos**: Diferentes arquiteturas, SO e linguagens distintas  
- **GRIP Computacional**
- **Arquitetura**: Ponto a Ponto ou Cliente-Servidor

## Objetivo
- Compartilhar recurso  
- Exclusão mútua  
- Relógio lógico e físico  

## Comunicação
- Sistemas distribuídos têm comunicação via **SOCKET**  
  - (IP, porta, máscara, objetos escritores/leitores)  
  - É **BLOQUEANTE**
- **Solução computacional em tempo de programação**: uso de **THREADS**

## Programação Multitarefas
- Uma **Thread** é um mini processo dentro de um processo
- Pode ser com **memória compartilhada**  
  - Necessita ser **sincronizada**
- Também pode ser **sem memória compartilhada**
- **Thread não faz comunicação compartilhada**, mas executa **concomitantemente**

# SISTEMAS PARALELOS

- **Homogêneos**: SO e linguagens de programação idênticas  
- **CLUSTER Computacional**
- **Arquitetura**: Ponto a Ponto  
  - Tolerância a falhas  
  - Segurança  
  - Escalável  
    - Capacidade de crescer o sistema  
    - Fazer um sistema correto  
  - Cabeamento de **fibra**

# Exercicios:

- **1- O que é um processo, como funciona?**
  - O processo representa o conjunto de recursos do sistema, memória, arquivos, CPU para execução de um programa
  - Ao abrirmos um jogo por exemplo ele vai criar um processo para aquele aplicativo
  - Possui um espaço de memoria reservado somente para ele
  - Ciclo de vida:
    - Criação
    - Execução
    - Espera (por eventos)
    - Finalização
  - Os processos PODEM se comunicar entre si com sockets ou memória compartilhada, mas por padrão são ISOLADOS

- **2- Thread o que é? Como funciona?**
  - A thread é parte de um processo mas que executa tarefas de forma idependente
  - Compartilha os mesmos recursos do processo (memória, arquivos)
  - Servem para executar varias tarefas dentro de um programa mas de diferentes tipos
    - Exemplo Navegador:
        - Estou olhando meu Instagram na PG1
        - Meu Youtube está tocando música na PG2
        - Facebook carregando na PG3
  - Threads tornam o programa rápido e eficiente pois permitem tarefas simultaneas.

- **3- Diferença entre processo e Thread**
  - Processos são ISOLADOS uns aos outros
  - Thread trabalham em CONJUNTO, unidas dentro de um mesmo processo
  - O processo é um programa com a execução no seu PRÓPRIO espaço de memoria
  - A thread é uma parte que está dentro mas trabalha de forma INDEPENDENTE no programa

- **4- Diferença entre Ponto a Ponto e Cliente Servidor**
    - Ponto a Ponto:
        - Os computadores se comunicam entre SI SEM um "REI" como se fosse um servidor central.
        - Um exemplo de Ponto a Ponto é o Torrent, que quando finalizamos a instalação ela fica semeando aos outros usuarios aquele arquivo
        - Fazendo assim o nosso computador uma peça do Ponto a Ponto sendo que podemos RECEBER e ENVIAR arquivos de qualquer maquina
    
    - Cliente Servidor:
        - Nessa arquitetura já temmos um ponto bem diferente que TEMOS o famoso "REI" que seria o SERVIDOR.
        - Ele vai fazer a distribuição de todos dados, arquivos saindo do próprio servidor aonde esta hosteando
        - Exemplo seria o nosso próprio site ou até mesmo o site do GitHub aonde ao colocamos a URL estamos solicitando ao servidor
        - Ao ser solicitado ao servidor ele vai responder com os dados para nós usuarios.
